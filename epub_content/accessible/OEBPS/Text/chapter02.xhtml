2. Rule Set

The current version of the Rule Set is 1.11.

eBook Conversion Specifications:

Input: Text PDF

Outputs: ePUB. If only ePub is delivered, it is optimized for display on different reading devices, including iPad and Kindle.

  1. File name should be PrintISBN_Version_format.file_extension format. Please find below the examples,
    • 978XXXXXXXXXX_1_epub_all.epub for the ePub Version which works on all devices including the ipad compatible.

  2. Cover jpg has to be added inside the ebook in order to display it as first page of the ebook. It should have at least 768 x 1024 pixel at 300 dpi. The original image ration has to be retained and the image scaled to either maximum height or maximum width. Additional margin around the cover image should be in white color; background color should be undefined. Applicable Formats: ePUB and Mobi

  3. Image resolution of 300 dpi should not be reduced. Maximum image quality is to be used. Applicable Formats: ePUB and Mobi

  4. Enclose images and captions and quotes and quote authors in <div class="group"> element.
    Images must have an <alt> attribute, preferably a short description. Alt="image" is not allowed, if a description is not possible, alt="" may be used. Images must be enclosed in an <div> that contains only the image source, no <p> tags, links or any other elements. Sizing is defined in the CSS.
    Images must contain less than 2 million pixels.
    Image captions must have the same alignment as the image.

  5. There should be no blank pages (i.e. after the cover image) in the eBook.
    Applicable Formats: ePUB and Mobi

  6. Halftitle (usually first page of frontmatter) should be ignored/removed in the eBook.
    Applicable Formats: ePUB and Mobi

  7. Title page (usually third page of frontmatter) is set as text using separate styles in CSS. Hyphenation should be suppressed. Styles are matched to PDF. Titlepage should follow immediately after cover. If Publisher is BoD Books on Demand, “Books on Demand” is added on title page in matching typeface.

    Applicable Formats: ePUB and Mobi

  8. The copyright page has to be placed directly after the text. Alignment of the copyright page is top-alignment. It should fit on one page, styles do not need to be matched to PDF.
    In the copyright, Books on Demand GmbH must contain a hyperlink to http://www.bod.de or to http://www.bod.fr, http://www.bod.dk, http://www.bod.se, http://www.bod.fi
    Applicable Formats: ePUB and Mobi

  9. Print ISBN number in copyright page should be replaced by eBook ISBN from provided in metadata information.
    Applicable Formats: ePUB and Mobi

  10. XHTML TOC (additional to NCX) should be included in the ebook. If chapter headers are present in the source file, but no toc, an xhtml toc is created. Only titles without headers or with more than 100 toc entries receive no html.toc
    Applicable Format: ePUB and Mobi

  11. Each chapter should be coded as a separate html file within the epub

  12. BoD standard CSS are to be used for all titles; unused elements are deleted from the CSS. Multiple CSS are used for different devices (iOS, kf8, rmsdk).

  13. Folder structure within the epub should be Fonts, Images, Misc, Styles & Text

  14. Language setting in content.opf must be main language of the book.

  15. PDF matching and ePub styling rules: General styling and layout are matched to the source PDF, but the following style rules apply to all epub output:
    1. Alignment: Headers must be left aligned or centered. Lists and bibliographies and similar appendices containing long hyperlinks must be left aligned. If necessary, soft hyphens are to be added to ensure proper line breaks. Excessive use of &nbsp; or pixel defined spaces for alignment are not allowed.
    2. TOC: No </br> are used within toc entries. If possible, a numbered TOC is coded as an ordered list <ol>.
    3. Top and bottom margins in excess of 3 em are not allowed for <p> elements.
    4. Font Size: The standard font used for <p> elements in each title is sized at 1 em by not specifying a font-size in the CSS.
    5. Tables shall be used for tabular data only.
    6. TOC shall not be coded as a table but preferably as a list.

  16. Special attention has to be paid to the correct display of special characters and umlauts, particularly in the system generated header on iPad and Sony devices. Characters/Fonts not supported by Apple, Kindle, Sony and other popular eReaders must be coded as scalable vector graphics (SVG) or using a licensed font provided by BoD. No other commercial fonts that require a license are to be embedded.

  17. References to material included only in print titles such as CDs on front- and backmatter are to be removed.

  18. If clearly commercial ads or links to major book retailers, e.g. amazon or kobo are contained, titles are to be flagged for BoD review.

  19. If a title contains tables with more 35 characters per row or more than 14 rows, they are coded as html and as an image using media queries defined in the standard CSS. If necessary, several zooming levels are captured for the image. Tables with less columns/characters are coded just as html. "colgroup"should be avoided

  20. Html coding shall not contain tags without function. Bold and italic should not be coded using span elements.

  21. Lists are to be formatted using <ul> and <ol> tags.

  22. The following are to be included in every file: Guide details for Mobi TOC, page-template.xpgt for ADE, display-options file for Apple. Applicable formats: ePuB and Mobi

  23. Standard TOC format should be followed for the frontmatter as mentioned below regardless where the PDF TOC is located (frontmatter or backmatter).
    Applicable Formats: ePUB and Mobi
    1. Cover image entry should not be included in TOC (both NCX and XHTML), although found as a part of the book.
      Applicable Formats: ePUB and Mobi
    2. TOC wording has to follow the headlines in the PDF; for standard items without headline in PDF see translation list.

  24. The start of the text has to be represented within TOC's (both NCX and XHTML), if the PDF content doesn't contain a chapter structure and headlines, to mark the beginning of the content (please use expression from translation list below). It should be referenced in the guide/reference as type="other.reader-start-page".

  25. There should be no space characters in front of question or exclamation marks (except for French texts).
    Applicable Formats: ePUB and Mobi

  26. The margins in the page-template.xpgt should be 1.25 em.
    Applicable Formats: ePUB

  27. Per Deutsche National Bibliothek requirement, the id element in ncx.toc must be a unique identifier and not identical to the corresponding play order element. For id, chapter-1 or navpoint-1 should be used.
