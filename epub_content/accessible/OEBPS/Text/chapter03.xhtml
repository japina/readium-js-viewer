3. Stylesheets

BoD CSS are to be used for all titles. Some styles in CSS, like drop cap styles, are applied to all titles, others are matched to PDF in accordance with BoD conversion rules. Additional styles and coding has to be commented in the CSS and follow understandable naming conventions. Styles not used for a specific title should be deleted from that title's CSS. Our goal is a structured archive with consistent styles and highly readable and easy to edit CSS.

Each html file has to contain links to all CSS. From the html file, devices are targeted by different media queries and directed to applicable styles in the corresponding CSS.

3.1 Default CSS "bod_standard.css"

BoD Default CSS contains all styles supported by all platforms as well as those for which we found a multi-platform solution. Current version is v2.

3.2 CSS for Kindle Devices "kf8.css"

Kindle devices require special coding and offer limited support of complex, CSS-based layout options. At this point, all Kindle devices can be targeted using @kf8 media queries. It is not possible to distinguish between one KF8 device and another, so the Kindle Fire applies the same styles as the Kindle eInk devices. With BoD standard CSS version 2, we discontinue support for mobi7, because amazon now offers an update to KF8 for all devices.

3.3 CSS for iOs devices "iOs.css"

Since Apple devices offer the best display options for tables and the widest support for epub3 features, we target them separately. On iphones, tables with less than 35 characters per row are displayed as html.

3.4 CSS for Small Displays and Apps "rmsdk.css"

For display in RMSDK-based reading apps and small eInk Readers, a seperate CSS is used. On all these devices, tables are displayed as images.
